2. Report Outputs: Fields, Definitions, and Formulas

Each report should be traceable to event codes, fields, formulas, and update frequency.
ReportFrequency / DimensionFieldsDefinition / FormulaRecommended View
Usage OverviewDaily / weekly / monthly; project x channelinteractions, sessions, valid questions, average turns, active devices, peak hoursValid questions = user messages excluding greetings, idle events, and system triggers.KPI cards + trend line + stacked channel chart
Need and Intent AnalysisDaily / weekly; project x intentintent, question count, share, growth, representative questions, related KB, source channelIntent share = intent question count / valid question count. Deduping rules must be explicit.Pareto bar + representative question table
Knowledge Coverage and GapsDaily / weekly; intent x KBKB hit rate, miss rate, low-score retrieval, uncovered questions, suggested KB topicsKB hit = retrieved score above threshold and used_in_answer = true. Miss = no result or fallback.Coverage matrix + gap queue
Answer QualityWeekly / monthly; character x intentpass rate, partial pass rate, error rate, low-confidence rate, review count, issue typePass rate = reviewed pass / reviewed total. Low confidence can use similarity or LLM judge confidence.Quality funnel + issue stacked bar
Channel and Device PerformanceDaily / weekly; channel x deviceentry opens, interactions, online rate, last heartbeat, average dwell time, abandonment rateOnline rate = devices with heartbeat / registered devices. Abandonment = unfinished or timed-out session.Channel mix + device health table
Retail Click and Conversation LinkageMinute / hourly / daily; project x UI element x sessionminute-level clicks, click-to-message window, click-before-question rate, answer-to-click rate, unlinked clicksClick linkage = clicks joined to conversation/message/response within the configured time window. Source DB timestamps should be kept at minute precision or better because GA imports cannot support this level of inspection.Click timeline + conversation-link table
Game and Machine Extension UsageDaily / weekly; project x device x game modulegame starts, completions, rewards, retries, scan actions, score/result distribution, follow-up questionsGame completion = completed game events / started game events. Follow-up question rate = sessions with game event and later user_message_sent / game sessions.Machine feature adoption matrix + game funnel
Voice and Multilingual ExperienceWeekly / monthly; judged language x channelAI-judged language distribution, STT success, TTS success, retry rate, foreign-language qualityLanguage should use AI judgment labels rather than treating raw detector probability as the final language. STT success = successful transcripts / voice input attempts.AI language judgment heatmap + voice reliability cards
Backend Test Resource UsageDaily / weekly; environment x test type x projecttest runs, pass/fail state, linked prompt/KB/response, actor role, production-exclusion flagProduction traffic should exclude backend_test_resource_recorded unless a report explicitly compares real usage and test usage.Backend resource table + test-result status cards
ROI and Service ValueMonthly; project x intentautomated questions, saved minutes, labor cost, AI cost, net value, improvement upsideNet value = estimated labor savings - LLM/STT/TTS cost. Assumptions must be adjustable.ROI waterfall + assumption panel

High-frequency gap

"How does the parking discount work?"
Action: add discount rules, limitations, and exceptions.

Low-confidence answer

"Are there events today?"
Action: connect a live event data source.

Voice interruption

Voice session interruption rate: 21%.
Action: inspect STT confidence and on-site noise.

Recommended implementation path:First define a unified event schema, then generate report tables or materialized views, and finally let the admin dashboard read aggregated data. Every dashboard block should show source events, formulas, and refresh frequency so stakeholders can connect user behavior to measurable outcomes and ROI.
