P0：確認客戶是否持續使用

先做

以最基礎且容易取得的系統事件，掌握客戶是否持續啟用 AI AMAZE、近期是否有流量，以及主要使用入口。

活躍專案 / 無互動專案

近 7 / 30 天是否有前台互動或 API 呼叫。

互動量與趨勢

每日、近 7 天、近 30 天訊息數或 session 數。

入口來源分布

Web、Kiosk、Robot、API、Mobile/Tablet 的使用量。

KB 最後更新時間

P1：確認使用體驗是否穩定

第二階段

在掌握使用量後，加入基礎品質訊號，協助辨識已使用但體驗可能受 KB、回應速度或語音穩定性影響的客戶。

KB 命中率

回答是否有使用知識庫；以命中比例作為基礎品質訊號。

兜底率 / 無答案率

AI 無法回答的比例；可先用 fallback 事件統計。

回應時間

平均與 P95 延遲，尤其適合 Kiosk / Robot / 語音場景。

STT / TTS 成功率

語音場景是否穩定；暫時只看成功 / 失敗事件。

後面做
