Action Queue · 代表性待處理項

dashboard 不只看數字，也要把可以行動的事排出來。

P1 · Trace 缺口補 RAG hit/miss 與 used chunk

否則 KB 覆蓋率與回答依據無法被後台追溯，只能看總使用量。

P2 · 品質標註建立人工 review 與 AI judge 欄位

讓高風險對話能進入待處理清單，而不是人工翻完整對話紀錄。

P3 · 入口健康把 entry opened 與 conversation started 分開

可判斷流量下降是需求、入口、裝置離線或體驗問題。
