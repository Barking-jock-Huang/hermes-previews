客戶產品使用狀態與 ROI 觀測 Dashboard 範例

本 Dashboard 用於彙整 AI AMAZE 客戶產品使用狀態，協助 PM、RD 與主管對齊產品觀測指標、資料紀錄需求與可視化優先順序。核心觀測面向包含：客戶是否持續使用、主要使用入口、知識庫與角色設定維護狀態、基礎回答品質、語音 / 載具 / 系統穩定性，以及後續可延伸的產品洞察。

資料來源jock@barkingdog.ai 可讀專案：林百貨、南門市場、壽山動物園；本頁只放彙總與示意，不含 token / 密碼。
頁面定位產品使用觀測概念頁，不是 CRM / CS 聯繫追蹤頁。
下一步用途作為 RD/PM 討論事件資料、彙總邏輯與視覺優先順序的範例。

ROI 模型初版

進階 ROI 不應只看互動量，需把「有效回答、人工替代、成本、改善成效」串起來。

1 使用需求有效 user question / 熱門意圖
2 回答品質KB 命中、低信心、兜底、錯誤
3 人工替代有效回答數 × 平均人工處理分鐘
4 成本估算token + STT/TTS + 載具維運
5 改善成效before/after pass rate 與未覆蓋問題下降
給 RD / PM 的資料記錄方向： 建議補齊 `conversation_started`、`message_sent`、`channel_type`、`device_id`、`kb_hit`、`kb_miss`、`fallback_triggered`、`response_latency_ms`、`stt_success`、`tts_success`、`user_feedback`、`review_label`、`cost_estimate` 等事件 / 欄位，讓目前可見的 dashboard 資料升級為可管理的產品觀測資料。

資料說明

本頁使用 jock@barkingdog.ai 可讀專案的彙總資料建立範例。互動量、語言、熱門詞、點擊、裝置數來自目前可讀統計 API；回答品質、ROI 與部分狀態欄位為下一階段資料設計示意，頁面中已用「待接 / 示意」標示。頁面不包含密碼、token、完整對話內容或個資。
